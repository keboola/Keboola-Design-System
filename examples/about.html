About Keboola Design System

Introduction

The Keboola Design System provides a consistent set of components, patterns, and guidelines for building Keboola applications. It ensures visual consistency, improves development efficiency, and creates a cohesive user experience across all Keboola products.

Core Principles

Use the gap-based spacing approach for consistent layouts
Follow the component hierarchy and proper nesting
Use design tokens consistently for colors, spacing, etc.
Maintain proper typography with provided classes
Keep components accessible with proper contrast and focus states

Key Features

Consistent Design Tokens Core
Colors, typography, spacing & more
Reusable Components Core
Buttons, cards, forms & more
Layout Patterns Core
Grid system & spacing guidelines

Getting Started

1. Add Required Dependencies

<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
<link href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css" rel="stylesheet">
<link href="dist/design-system.css" rel="stylesheet">

2. Set Up Basic HTML Structure

Every page should follow the basic structure with proper container classes and spacing.

3. Explore Example Pages

Check out the example pages in the examples directory to see the design system in action:

  • login-register.html: Authentication forms with validation
  • people-management.html: User management interface
  • transportation-dashboard.html: Data dashboard layout
